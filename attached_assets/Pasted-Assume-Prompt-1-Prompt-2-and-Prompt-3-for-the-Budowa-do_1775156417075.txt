Your task now is to implement the "House Build Timeline + Stage Budget + Team Planning" layer for the Poland-first House Build module.

GOAL
Create a practical investor-oriented planning layer that helps the user:
- understand the likely order of house-building stages
- estimate stage-by-stage budget structure
- plan when to involve professionals
- track dependencies and delays
- connect future contractor-search logic to build stages
- keep everything separate from strict formal/legal path logic

IMPORTANT PRODUCT RULES
- This is not a bank-grade cost estimator and not a contractual quote tool.
- Budgets must be presented as planning estimates, ranges, or placeholders for investor organization.
- The module must not promise exact final construction cost.
- Labor and material amounts for house-building stages should be designed as reference planning blocks, not as final execution quantities unless already backed by project-specific data.
- The app should help the user think in stages, dependencies, and decisions.
- Keep beginner-friendly language.

BUILD THESE FEATURES

1. HOUSE BUILD TIMELINE MODULE
Create a dedicated timeline/planning layer inside the House Build module.

The timeline should support:
- project start planning
- stage ordering
- estimated stage duration placeholders
- dependency chains
- blocked / waiting / ready states
- delay notes
- milestone markers
- stage completion tracking

Create a timeline view and a stage list view.

2. DEFAULT STAGE ORDER
Use the existing build stages and connect them into a planning sequence.

At minimum support this default order:
- Plot and site preparation
- Foundations and foundation waterproofing
- Structural shell / walls
- Roof structure and covering
- Windows and exterior doors
- Internal rough installations
- Insulation / airtightness / energy stage
- Facade and external finishing
- Internal finishing and fit-out
- Completion readiness

Allow the user to:
- reorder only where safe/custom planning is needed
- keep the default recommended order visible
- see dependency warnings if they try to move a stage too early

3. STAGE DEPENDENCY LOGIC
Add stage dependency rules.

Examples:
- foundations should not be marked ready to close until key checks are complete
- structural shell should precede roof closure
- windows/exterior doors should not appear too early in the sequence without shell/roof context
- rough installations should not be planned after interior finish completion
- internal finishing should be blocked until prerequisite wet/rough stages are sufficiently complete
- completion readiness depends on earlier stages and formal completion flow

Keep this practical, not over-engineered.

Create structured entities/types/schemas such as:
- HouseBuildBudget

- BudgetAssumption
- BuildTimelineMilestone

Each stage should support estimated cost sections like:

- user custom note

Important:
- do not import hardcoded market prices yet in this step
- this step prepares the architecture and investor planning UX
