Your task now is to implement the Poland-first formalities and legal-path planning layer for the "Budowa domu" module.

GOAL
Help the user understand which formal path they may need to follow when planning a single-family house in Poland, what documents are typically required, what must happen before works begin, and what formal closing steps may be needed.

IMPORTANT RULES
- This module must be informational and planning-oriented, not legal advice.
- Do not present one path as universally correct for every case.
- Always show a note that the investor should verify the final path with the local authority, designer, and project documentation.
- Keep the wording beginner-friendly.
- Preserve a strong separation between:
  - official/formal process guidance
  - technical construction guidance
- Store official-source metadata for each rule or checklist block.

BUILD THESE FEATURES

1. FORMAL PATH WIZARD
Create a step-by-step wizard that asks:
- Is there an MPZP for the plot? (yes / no / unknown)
- If no or unknown, does the user already have WZ? (yes / no / unknown)
- Is the house a single-family house?
- Is it free-standing?
- Planned footprint area
- Number of above-ground floors
- Is the build for own housing purpose?
- Is the user a first-time investor?
- Does the user want the app to show a more conservative recommended path?

2. PATH OUTPUT LOGIC
Based on the answers, show one of the following planning paths:
- Building permit path
- Notification with building design path
- Simplified <=70 m2 path (only when conditions appear to match)

Important:
- For the <=70 m2 path, add a clear caution note that the simplified path is not always the best practical choice for first-time investors
- If the user marks “first-time investor”, the app may recommend the more conservative path explanation
- Do not claim legal certainty; explain likely path and required verification

3. OFFICIAL PROCESS CHECKLISTS
Add official-style checklist content for:
- checking MPZP
- obtaining WZ if MPZP does not apply
- preparing design documentation
- preparing the declaration of right to use the property for building purposes
- collecting required opinions/agreements/permits if applicable
- confirming designer authorization / professional entitlement documentation
- using e-Budownictwo for online filing
- preparing for start of works
- construction log / EDB
- completion notice or occupancy permit

4. BEFORE-WORKS CHECKLIST
Create a “Before works begin” section that includes planning checks such as:
- geodetic staking out of the house on the plot
- terrain leveling / surveying and site preparation
- arranging temporary or target power and water access
- organizing the site (for example fencing/site preparation/worker facilities as appropriate)
- planned start date notification flow

Do not turn this into a heavy contractor manual.
Keep it investor-oriented.

5. START-OF-WORKS FORMAL STEP
Add a dedicated section for:
- planned start-of-works notification
- required basic data
- reminder that works should not start before the indicated start date process is completed
- e-Budownictwo compatibility
- source metadata

6. ELECTRONIC CONSTRUCTION LOG (EDB)
Create a section for the electronic construction log:
- what it is
- who uses it
- that the investor can apply for it online
- that participants such as site manager / designer / inspector can be added
- that it can be submitted at the end of the build
- keep this practical and simple
